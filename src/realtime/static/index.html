Framework

MITRE ATT&CK for ICS

MITRE ATT&CK for ICS is a knowledge base of adversary tactics, techniques, and procedures targeting Industrial Control Systems. It extends the enterprise ATT&CK matrix to cover operational technology (OT) environments including SCADA, PLCs, RTUs, and field devices in sectors such as energy, water, and transportation.

Unlike enterprise techniques, ICS techniques describe attacks that can cause physical consequences — equipment damage, safety failures, or disruption of real-world processes. The matrix covers 12 tactic categories aligned to the ICS kill chain.

In this simulation

EV charging station attacks are mapped to the ICS matrix. The primary technique used is T0836 — Modify Parameter under the Impair Process Control tactic, demonstrated with a real-time CUSUM + IQR residual detector.

T0836

Modify Parameter

T0836 — Modify Parameter describes adversary actions that alter setpoints, thresholds, or configuration values on controllers to cause unintended physical behaviour. In this simulation, the attacker adds a constant offset θ (in equivalent kWh-minutes) to the reported pilot current signal.

Tactic: Impair Process Control (TA0106)

The adversary is not seeking persistence but is actively manipulating the physical process — in this case inflating energy delivery readings to cause overbilling or mask battery degradation.

Attack Class

False Data Injection (FDI)

A False Data Injection attack corrupts sensor measurements or telemetry before they reach the control system or billing platform. In EV charging, the target is typically the pilot current signal or energy meter register. By inflating these values an attacker can cause overbilling, hide battery damage, or generate fraudulent demand signals to grid operators.

Why FDI evades simple detectors

Static threshold detectors fail because FDI attacks can be calibrated to stay within expected measurement variance. Model-based detectors like NARX and BiLSTM compare measurements against a learned dynamic model, making injections far harder to conceal.

Metric

Error-on-Estimation (EoE)

At each timestep the model produces a one-step-ahead prediction of the pilot current. The EoE is the absolute residual between the observed (possibly injected) measurement and that prediction.

Definition

EoE(t) = | y(t) − ŷ(t) |

Under normal conditions EoE hovers near zero. Under a T0836 attack it rises proportionally to the injection magnitude θ. The chart plots this signal in real time — blue dots are clean, amber indicates active attack, red indicates a confirmed detection.

Detector

CUSUM — Cumulative Sum

CUSUM accumulates deviations from a reference level over time, making it sensitive to sustained small shifts that point-in-time detectors miss.

Algorithm

S(t) = max(0, S(t−1) + EoE(t) − μ)

An alarm fires when S(t) ≥ 5σ of the clean EoE distribution. CUSUM is the primary detector — it reliably catches attacks with θ ≥ 5 within a few steps.

Detector

IQR — Interquartile Range

The IQR detector flags outlier EoE values using a non-parametric upper bound.

Algorithm

UB = Q3 + 3 × IQR

An alarm fires when 3 consecutive timesteps exceed UB. IQR complements CUSUM — it activates immediately on large-magnitude injections (θ ≥ 30) where a single step already generates a massive EoE spike.
