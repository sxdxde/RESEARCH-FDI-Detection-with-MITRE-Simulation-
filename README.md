# NARX + Attention-BiLSTM FDI Detection with Adaptive Two-Stage IF–CUSUM

Extension and improvement of the base paper:
> **"False Data Injection Attack Detection in EV Charging Network Using NARX Neural Network"**
> Basumatary, Khatua, Nath — IEEE Transactions on Transportation Electrification, Vol. 11, No. 4, Aug. 2025.

This project trains two neural energy estimators (NARX and Attention-BiLSTM) on the public **ACN-Data-Static** dataset and detects low-amplitude FDI attacks via an **adaptive two-stage Isolation Forest + CUSUM** detector. Final results: **NARX F1 = 97.9% / Recall = 99.9%**, **BiLSTM F1 = 97.0% / Recall = 98.7%** on realistic attacks scaled at 1–2× the mean session delivery.

---

## Key Contributions over the Base Paper

| Contribution | Description |
|---|---|
| **Scale of dataset** | ACN-Data-Static (85K+ sessions, 3 sites) vs. small private dataset |
| **Low-amplitude attack regime** | φ ∈ [1, 2]× mean delivery vs. 8–15× (trivially detectable) |
| **Second neural estimator** | Attention-BiLSTM (2.16 M params, Bahdanau attention) alongside NARX (171 params) |
| **4-D Isolation Forest features** | [ε, ε², \|Δε\|, z_loc] vs. 1-D raw EoE — breaks IF/CUSUM degeneracy |
| **Distribution-shift cap** | h_cap = 3.5 × p99(EoE_clean) — recovers 29 recall points over unconstrained F1-tuning |
| **Recall-priority calibration** | Progressive precision-floor fallback (0.90 → 0.70) before pure F1-max |

## Final Results (ACN-Data-Static, φ ∈ [1, 2])

| Model | Accuracy | Precision | Recall | F1 |
|---|---|---|---|---|
| **NARX + IF–CUSUM** | **99.57%** | **96.0%** | **99.9%** | **97.9%** |

| Parameter | Value |

## Attack Model

```
ỹ(t) = y(t) + φ · μ₊,    φ ~ Uniform[1, 2]
```

Where `μ₊` is the mean positive delivery in the session. This injects a 25–50% billing overcharge per attacked timestep, calibrated to be physically plausible (≤ 2 kWh per 5-min interval) while remaining below the gross-error threshold of rule-based detectors.

- **Injection fraction:** 10% of all estimation timesteps chosen uniformly at random
- **Total attack samples:** 33,378 out of 333,783 timesteps

---

## Detection Pipeline

### Stage 1 — 4-Dimensional Isolation Forest
Feature vector per timestep:
```
f(t) = [ε(t),  ε(t)²,  |Δε(t)|,  z_loc(t)]
```
- `ε(t)` — raw EoE magnitude
- `ε(t)²` — squared EoE (amplifies contrast)
- `|Δε(t)|` — one-step EoE increment (captures sudden FDI spikes)
- `z_loc(t)` — local z-score in a rolling 20-step window

IF trained with 200 trees, contamination=0.12, 15,000-point subsample.

### Stage 2 — Adaptive CUSUM with Distribution-Shift Cap
```
S(t) = max(0, |ε(t)| - k)
alarm if S(t) ≥ h,   where h ≤ h_cap = 3.5 × p99(EoE_clean)
```
Threshold `h` is selected by **recall-priority calibration**: maximise validation recall subject to precision ≥ 0.90, with the floor progressively relaxed to 0.70 before falling back to F1-maximisation.

### Combined Decision
```
δ(t) = δ_IF(t) AND δ_CUSUM(t)
```
The AND gate suppresses IF false alarms (caused by contamination=0.12) while preserving recall from both stages.

## Dataset

**ACN-Data-Static** — [https://ev.caltech.edu/dataset](https://ev.caltech.edu/dataset)

- 85,000+ EV charging sessions from 3 California sites (Caltech, JPL, Office-01)
- 5-minute energy delivery intervals
- Session fields: `sessionID`, `siteID`, `connectionTime`, `disconnectTime`, `doneChargingTime`, `kWhDelivered`, `userRequestedEnergy`, `minutesAvailable`
- Train/test split: 10,000 / 5,000 sessions, **zero session overlap**
